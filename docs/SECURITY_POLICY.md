# セキュリティポリシー

このワークスペースは、Claude Codeを安全に使うための**3層防御**で守られています。

---

## 3層防御とは

### 第1層: サンドボックス（OSレベルの制限）

一番強い壁です。Claude Codeが動くときに、OSが自動的にファイルやネットワークへのアクセスを制限します。

- プロジェクトフォルダ外のファイルへの書き込みを制限
- 許可されていないネットワーク通信を制限

**設定場所**: `.claude/settings.json` の `sandbox` セクション

### 第2層: 権限制御（deny/allowリスト）

Claude Codeに「これはやっていい」「これはダメ」と伝えるリストです。

**禁止されていること（deny）:**
- 危険なコマンド（ファイル削除、権限変更など）
- ネットワーク通信コマンド（curl, wget, ssh など）
- 機密ファイルの読み取り・編集（.env, 秘密鍵など）
- 本番環境の操作（AWS, Terraform など）
- 権限昇格（sudo, su）

**許可されていること（allow）:**
- ファイルの閲覧（ls, cat, head など）
- Git操作（status, diff, log, commit, push）
- ディレクトリ・ファイルの作成（mkdir, touch）
- Web検索・Web取得

**設定場所**: `.claude/settings.json` の `permissions` セクション

### 第3層: hookスクリプト（最終防衛線）

Claude Codeがコマンドを実行する前に、スクリプトが自動でチェックします。
第2層をすり抜けるような複雑なパターンもここで止めます。

**用意されているスクリプト:**

| スクリプト | タイミング | 役割 |
|-----------|-----------|------|
| `block-dangerous-commands.sh` | 実行前 | 危険なコマンドパターンを検出してブロック |
| `block-main-push.sh` | 実行前 | main/masterブランチへの直接pushをブロック |
| `protect-sensitive-files.sh` | 実行前 | 機密ファイルへのアクセスをブロック |
| `protect-data.sh` | 実行前 | 大きなファイルやデータファイルの読み込みを警告 |
| `scan-tool-output.sh` | 実行後 | 外部データに含まれるインジェクション攻撃を検出 |

**設定場所**: `.claude/scripts/` ディレクトリ

### プロンプトインジェクション対策

外部から取得したデータ（Webページ、Slackメッセージ、Notionページ等）には、
AIを騙して意図しない操作をさせようとする「プロンプトインジェクション」が含まれている可能性があります。

このテンプレートでは以下の対策を実装しています:

| 攻撃パターン | 対策 |
|------------|------|
| `curl | bash` 等のリモートコード実行 | `block-dangerous-commands.sh` でブロック |
| Base64エンコードによる難読化 | 同上（`base64 -d \| bash` パターンを検出） |
| 環境変数・機密情報の外部送信 | 同上（`env \| curl` パターンを検出） |
| システムプロンプト窃取 | 同上（`CLAUDE.md` の外部送信パターンを検出） |
| 外部データ内の悪意ある指示 | `scan-tool-output.sh` で警告（PostToolUse） |
| MCP経由の全文検索による情報漏洩 | `settings.json` でsearch系ツールをdeny |

参考: [AIエージェントの3層プロンプトインジェクション対策](https://creators.bengo4.com/entry/2026/03/24/080000)

## 変更しないでください

以下のファイルはセキュリティの根幹です。変更するとワークスペースの安全性が損なわれます。

- `.claude/settings.json`
- `.claude/scripts/*.sh`

設定の変更が必要な場合は、管理者に相談してください。

---

## 困ったときは

- 「このコマンドは実行できません」と表示された → セキュリティ制限です。安全な別の方法を試してください
- 機密ファイルにアクセスできない → 意図的な制限です。直接ファイルを開いて確認してください
- main/masterにpushできない → ブランチを切って作業してください
