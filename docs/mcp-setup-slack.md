# Slack MCP セットアップガイド

Slack公式のリモートMCPサーバーを使って、Claude CodeからSlackを操作できます。
Bot不要、OAuth 2.0認証のみで利用可能です。

## セットアップ

```bash
claude mcp add-json slack '{"type":"http","url":"https://mcp.slack.com/mcp","oauth":{"clientId":"1601185624273.8899143856786","callbackPort":3118}}'
```

## 認証手順

1. Claude Code で `/mcp` を実行
2. Slack の MCP サーバーで「Authenticate」を選択
3. ブラウザが開くので、ワークスペースでSlackにログインして認可
4. 「Authentication successful」と表示されたら完了

## 認証状態の確認

`/mcp` を実行すると接続状態を確認できます。

- `Status: ✔ connected` — 正常に接続中
- `Status: ✘ not authenticated` — 認証が必要（上記手順を実行）

## できること

| 機能 | 説明 |
|------|------|
| メッセージ検索 | チャンネルのメッセージを検索 |
| チャンネル履歴 | 指定チャンネルの最新メッセージを確認 |
| スレッド読み取り | URLを渡してスレッド全体を取得 |
| メッセージ送信 | チャンネルやDMにメッセージを送信（確認あり） |
| キャンバス操作 | Slackキャンバスの読み取り・作成 |
| ユーザー検索 | ユーザーのプロフィール情報を検索 |

## 使い方の例

```
「#general の最新メッセージを見せて」
「このスレッドを読んで要約して: https://slack.com/archives/C0268xxx/p1771xxx」
「#dev-data に『デプロイ完了しました』と投稿して」（送信前に確認あり）
```

## セキュリティルール

- **全文検索は使わない** — チャンネルを指定して読み取る
- **URLを渡して操作する** — 「全部見て」とは言わない
- **送信前に必ず確認** — Claude Codeが文面を提示してから送信

## FAQ

- **Bot不要。** ユーザートークンで動作
- **自分が参加しているチャンネル・DMのみ** アクセス可能
- Appのチャンネル招待不要
- 他プロジェクトでも使いたい場合: 上記コマンドをそのプロジェクトで実行するか、グローバル設定に追加
