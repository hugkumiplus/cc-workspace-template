# 初回セットアップ

## 概要

ワークスペースを自分の業務に合わせてカスタマイズする。
会社・ブランド情報をヒアリングし、CLAUDE.mdを自動生成する。

## ワークフロー

### Step 1: 既存のCLAUDE.mdを確認（Automatic）

プロジェクトルートの `CLAUDE.md` を読み込む。
既にカスタマイズ済み（以下の2条件を両方満たす場合）:
- 「## あなたの会社について」セクションが存在する
- そのセクション内に具体的な会社名が記載されている（テンプレートのままではない）

上記を満たす場合:

> CLAUDE.md は既にカスタマイズされています。
> 再セットアップしますか？ 上書きされます。

### Step 2: 会社情報のヒアリング（Interactive）

`AskUserQuestion` で対話的にヒアリング。親しみやすいトーンで。

#### Step 2a: 会社名

> ワークスペースのセットアップを始めましょう！
>
> まず、あなたの会社名を教えてください。

#### Step 2b: 事業内容

> ありがとうございます！次に、どんな事業をしていますか？
>
> 例: D2C通販、SaaS、コンサルティング、メディア運営、製造業 など

#### Step 2c: ブランド・サイトURL

> あなたの担当するブランドやサービスのWebサイトURLを教えてください。
> （複数ある場合は全部教えてください）
>
> 例: https://www.example.com/

#### Step 2d: あなたの役割

> あなたの部署や役割を教えてください。
>
> 例: マーケティング担当、EC運営、CS、経理、制作、管理職 など

### Step 3: Webサイトの分析（Automatic）

Step 2c で渡されたURLを `WebFetch` で読み取り、以下を分析する:

- ブランドのトーン（言葉遣い、雰囲気）
- 主要商品・サービス
- ターゲット層
- 競合との差別化ポイント

分析結果をユーザーに提示:

> サイトを分析しました:
> - **ブランドトーン**: {分析結果}
> - **主要商品**: {分析結果}
> - **ターゲット**: {分析結果}
>
> この内容でCLAUDE.mdを生成してよいですか？ 修正があれば教えてください。

### Step 4: CLAUDE.md の生成（Automatic）

以下の構造で CLAUDE.md を生成する。元のテンプレートの原則・ワークフロー部分は維持し、
会社固有の情報を追加する。

```markdown
# {会社名} AI ワークスペース

## このワークスペースについて
{会社名}の業務を Claude Code で効率化するためのワークスペースです。

## あなたの会社について
- {事業内容}
- {ブランド情報（サイト分析結果から）}

### ブランドトーン
- {ブランドごとのトーン}

## あなたの役割
- {部署・役割}

## 業務ルール
- {業界固有のルール（例: D2Cなら薬機法注意）}
- Notion操作: 共有DBへの書き込みは必ず確認を取ること
- Slack操作: 送信前に必ず文面を確認

{以下、テンプレートの原則・ワークフロー部分をそのまま維持}
```

### Step 5: 確認と保存（Interactive）

生成した CLAUDE.md をユーザーに表示し、確認を取る。

> CLAUDE.md を生成しました:
> [内容を表示]
>
> これでよろしいですか？ 修正したい箇所があれば教えてください。

OKが出たら CLAUDE.md に書き込む。

### Step 6: 完了メッセージ（Automatic）

> セットアップ完了！ 🎉
>
> CLAUDE.md があなたの業務に合わせてカスタマイズされました。
> Claude Code を再起動（`/clear`）すると、新しい設定が反映されます。
>
> 次のステップ:
> - `/task-start` でタスクを始める
> - `/skill-create` で自分だけのスキルを作る
> - `/company` でAIチームを構築する

## 業界別の自動判定ルール

ヒアリング結果から業界を判定し、業界固有のルールを自動追加する:

| 業界 | 自動追加するルール |
|------|------------------|
| D2C通販・化粧品・健康食品 | 薬機法・景表法に注意。NG表現を含む出力は必ず確認を求める |
| SaaS | 技術ドキュメントはMarkdownで。APIの破壊的変更に注意 |
| メディア | 著作権・引用ルールの遵守。ファクトチェックを必ず行う |
| コンサルティング | クライアント機密情報の取り扱いに注意 |
| 製造業 | 品質管理基準・ISO規格への準拠を意識 |
| その他 | 汎用的なルール（確認してから実行、機密情報に注意） |

## 重要な注意事項

- CLAUDE.md のテンプレート部分（原則、ワークフロー・オーケストレーション、セキュリティ等）は維持する
- 会社固有の情報のみを追加・カスタマイズする
- サイト分析が失敗した場合はスキップし、手動入力で進める
- 既存の CLAUDE.md がカスタマイズ済みの場合は上書き確認を取る
