# CC Workspace Template

## このワークスペースについて

非エンジニアでも安全に Claude Code を業務利用できるテンプレートです。
セキュリティは **3層防御**（サンドボックス・権限制御・hookスクリプト）で守られています。
詳細は `docs/SECURITY_POLICY.md` を参照してください。

---

## セキュリティ方針

事故防止のために意図的な制限を設けています。
「なぜ制限するのか」を理解し、制限の回避や無効化を行わないでください。

### 機密情報の保護

- `.env`、秘密鍵（`.pem`, `.key`）、認証情報を読み取らない
  - **理由**: 誤って表示・出力されると情報漏洩につながり、取り返しがつかない
- パスワード、APIキー、トークンをログや出力に含めない
- 機密情報をコードにハードコードしない（環境変数を使用する）

### 破壊的操作の防止

- 広範囲なファイル削除は実行しない
  - **理由**: 一度消えたデータは復元できない
- パーミッションの危険な変更は実行しない
  - **理由**: 全ユーザーに書き込み権限が付与され、セキュリティが崩壊する
- `eval`, `exec` による動的コマンド実行は行わない
  - **理由**: 任意のコードが実行される入口になる

### ネットワーク通信の制御

- `curl`, `wget` による外部通信は行わない
  - **理由**: 意図しないデータ送信（情報漏洩）を防ぐ
- 外部への通信が必要な場合は `WebFetch` ツールを使用する
- ユーザーの許可なくネットワーク通信を発生させない

### 本番環境の保護

- 本番データベースへの直接接続・変更を行わない
- 本番サーバーへの直接デプロイを行わない（CI/CDを通すこと）
- クラウドサービス（AWS, Terraform等）の直接操作は行わない

### データ保護

- **大きなファイル（1MB超）をそのまま読み込まない** — 必要な部分だけ抽出する
- **データファイル（CSV, Excel, JSON等）は注意して扱う** — 個人情報が含まれていないか確認する
- 生データをチャットに貼り付けない — ファイル操作ツールを通じて処理する
- 個人情報（メールアドレス、電話番号、住所等）は匿名化・マスキングしてから処理する
  - **理由**: 生データがAIの入力に含まれると、意図しない情報漏洩のリスクがある

## Git操作ルール

- コミットメッセージは**日本語**で、変更内容がわかるように書く
- `main`/`master` ブランチへの直接pushは禁止（ブランチを切って作業する）
- `git push --force` は禁止（履歴が消えて復元できなくなる）
- `git add .` / `git add -A` は禁止（意図しないファイルがコミットされる恐れがある）
- コミット前に `git diff` で変更内容を確認する

---

## コード品質の方針

- SQLインジェクション、XSS等のセキュリティ脆弱性を含むコードを書かない
- エラーメッセージに機密情報（DB接続文字列、APIキー等）を含めない
- エラーハンドリングを適切に行う

---

## ファイル操作の方針

- 既存ファイルを編集する前に必ず内容を確認する
- 新規ファイル作成は必要最小限にする
- バックアップなしに大量のファイルを変更しない
- `.claude/` 配下の設定ファイルやスクリプトは変更しない

---

## チーム機能の活用

Claude Code のチーム機能を使うと、複数のエージェントを並列で稼働させることができます。

- **チーム作成**: 秘書やカンパニースキルと連携し、部署ごとのエージェントをチームとして構成できる
- **並列処理**: リサーチ・ドキュメント作成・コードレビューなど、独立したタスクを同時に進行
- **セットアップ**: `/secretary` で初回セットアップ時にチーム機能の案内を受けられる

---

## 言語設定

- ユーザーとの対話は日本語で行う
- コミットメッセージは日本語で書く
- コード内のコメントはプロジェクトの慣習に従う

---

## スキルの利用

`.claude/skills/` 配下に以下のスキルが利用可能です:

- `/secretary` — パーソナル秘書（エントリーポイント。TODO管理、壁打ち、メモ）
- `/company` — 仮想会社組織（秘書が窓口、部署に振り分け）
- `/skill-create` — 新しいスキルを対話的に作成・改善

各スキルのSKILL.mdに従って実行してください。
